# أفضل البرومبتات للبرمجة مع Manus AI

هذه قائمة بأفضل البرومبتات وقوالب جاهزة للنسخ لمساعدتك في الحصول على أفضل النتائج عند كتابة الأكواد البرمجية باستخدام Manus AI.

---

## 1. طلب كود نظيف (Clean Code)

**الهدف:** الحصول على كود يتبع مبادئ الكود النظيف، مع أسماء متغيرات واضحة وتعليقات مفيدة.

**البرومبت:**
```
أريد دالة بايثون باسم `calculate_user_age` تأخذ تاريخ الميلاد بصيغة `YYYY-MM-DD` كمدخل. **الرجاء التأكد من أن الكود يتبع مبادئ Clean Code، مع أسماء متغيرات واضحة، وتعليقات فقط عند الضرورة، وتقسيم المنطق المعقد إلى دوال مساعدة صغيرة.**
```

---

## 2. تطبيق مبادئ SOLID

**الهدف:** بناء برمجيات قوية وقابلة للصيانة من خلال تطبيق مبادئ SOLID الخمسة.

**برومبتات لكل مبدأ:**

*   **Single Responsibility (SRP):**
    ```
    قم بإنشاء كلاس `ReportGenerator` مسؤوليته الوحيدة هي إنشاء التقارير. أي منطق لمعالجة البيانات يجب أن يكون في كلاس منفصل باسم `DataProcessor`.
    ```
*   **Open/Closed (OCP):**
    ```
    صمم نظام الإشعارات هذا بحيث يكون مفتوحًا للإضافات (مثل إضافة قنوات إشعار جديدة كـ Slack) ومغلقًا للتعديل على الكود الأساسي. استخدم استراتيجية التصميم (Strategy Pattern).
    ```
*   **Liskov Substitution (LSP):**
    ```
    لدينا كلاس أساسي `Bird`. عند إنشاء كلاسات فرعية مثل `Penguin` و `Sparrow`، تأكد من أن الكلاس الفرعي يمكن أن يحل محل الكلاس الأساسي دون التأثير على سلوك البرنامج.
    ```
*   **Interface Segregation (ISP):**
    ```
    لدينا واجهة `Worker` تحتوي على دوال للعمل والأكل والنوم. افصل هذه الواجهة إلى واجهات أصغر (`IWorkable`, `IEatable`) لتجنب إجبار الكلاسات على تنفيذ دوال لا تحتاجها.
    ```
*   **Dependency Inversion (DIP):**
    ```
    عند بناء كلاس `UserController` الذي يعتمد على `MySQLDatabase`، استخدم حقن التبعية (Dependency Injection) واجعل `UserController` يعتمد على واجهة `IDatabase` بدلاً من التنفيذ الملموس.
    ```

**الهدف:** حل المشاكل البرمجية المتكررة باستخدام حلول مجربة وفعالة.

**البرومبت:**
```
أحتاج إلى إدارة إعدادات التطبيق في مكان واحد. **استخدم نمط التصميم Singleton لإنشاء كلاس `AppSettings` يضمن وجود نسخة واحدة فقط منه على مستوى التطبيق.**
```

## 4. بناء تطبيقات ويب (Web Development)

**الهدف:** إنشاء تطبيقات ويب متكاملة باستخدام أطر عمل حديثة.

**برومبتات أمثلة:**

*   **Python/Flask:**
    ```
    باستخدام أداة `file`، قم بإنشاء تطبيق Flask بسيط في ملف `app.py`. يجب أن يحتوي التطبيق على نقطة نهاية `/api/v1/users` تستجيب بقائمة من المستخدمين بصيغة JSON. قم بتضمين مستخدمين وهميين كمثال.
    ```
*   **TypeScript:**
    ```
    اكتب دالة TypeScript باسم `formatDate` تأخذ كائن `Date` كمدخل وتعيد التاريخ كسلسلة نصية بصيغة `DD/MM/YYYY`. تأكد من إضافة الأنواع (types) بشكل صحيح للمدخلات والمخرجات.
    ```
*   **React/Node.js:**
    ```
    أريد بناء تطبيق قائمة مهام (Todo App). ابدأ بإنشاء الواجهة الخلفية باستخدام Node.js و Express، مع نقطة نهاية `POST /todos` لإضافة مهمة جديدة. ثم، باستخدام React، قم ببناء مكون واجهة أمامية يحتوي على حقل إدخال وزر لإرسال المهمة إلى الواجهة الخلفية.
    ```
